简介

这是一个基于WebComponent、Scss、Spanner以Gulp为打包工具的Starter。

通过在pages下新建html，entries下新建页面主Javascript，styles下新建样式文件来新建入口。

本页大概的HTML模板如下：

<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">
<title>WCUI-Starter</title>
<link rel="stylesheet" href="/styles/reset.css">
<link rel="stylesheet" href="/styles/components/index.css">
<script src="/assets/spanner.js"></script>
<script src="/scripts/utils.js" defer></script>
<script src="/components/index.js" defer></script>
</head>
// ........
</html>

页面Javascript模板如下：

document.addEventListener('DOMContentLoaded', async function () {
    //wait for component loaded
    window.spanner.loadWebComponents(['button', 'selector','input','loading','dialog','dateTimePicker']).then(() => {
        // write your code ...
    })
})

加载器组件

属性：

  • successtext: 加载成功时显示的文本
  • failuretext: 加载失败时显示的文本

内部属性：

  • status: 获取加载器当前状态

方法：

  • switchLoading: 设置加载器为加载中。
  • switchSuccess: 设置加载器为成功，返回一个标志加载器完成成功动画的Promise。
  • switchFailure: 设置加载器为失败，返回一个标志加载器完成失败动画的Promise。

示例：

代码：

<wc-loading status="loading" id="test-loading" successtext="success" failuretext="failure"></wc-loading>


<button onclick="document.querySelector('#test-loading').switchLoading()">Loading</button>
<button onclick="document.querySelector('#test-loading').switchSuccess().then()">Success</button>
<button onclick="document.querySelector('#test-loading').switchFailure().then()">Failure</button>

输入框组件

属性：

  • type: 支持text,number。分别设置不同的输入框类型
  • size: 支持small,normal,big。分别表示不同的输入框大小
  • value: 设置输入框的值。
  • required: 设置输入框是必填。（验证器）
  • maxlength: 设置输入框输入的最大长度。（验证器）
  • minlength: 设置输入框输入的最小长度。（验证器）
  • min: 设置输入框输入的最小数字。（验证器）
  • max: 设置输入框输入的最大数字。（验证器）
  • regex: 设置输入框输入的正则匹配。（验证器）
  • form: 设置输入框输入的父级form。（验证器）
  • name: 设置输入框的输入元素名，与form沟通使用。
  • disabled: 设置输入框的是否禁用。
  • preventclean: 设置输入框的是否禁用来自表单的clear请求。

内部属性：

  • value: 输入框的当前值。

Slot：

  • label: 输入框的标签。

示例：

label

代码：

<wc-input type="text" maxlength="12" required="true" style="width: 250px;display: block;" value="示例"> <span slot="label">label</span> </wc-input>

弹出框组件

属性：

  • type: 支持custom,prompt,alert三种弹出框。分别设置不同的弹出框行为
  • prompt-value: 设置输入框的值。
  • prompt-required: 设置输入框是必填。（验证器 - prompt类型弹出框）
  • prompt-maxlength: 设置输入框输入的最大长度。（验证器 - prompt类型弹出框）
  • prompt-minlength: 设置输入框输入的最小长度。（验证器 - prompt类型弹出框）
  • prompt-min: 设置输入框输入的最小数字。（验证器 - prompt类型弹出框）
  • prompt-max: 设置输入框输入的最大数字。（验证器 - prompt类型弹出框）
  • prompt-regex: 设置输入框输入的正则匹配。（验证器 - prompt类型弹出框）
  • prompt-disabled: 设置prompt类型弹出框的输入框的是否禁用。

事件：

  • PromptOk: prompt类型弹出框的确认事件，表示用户确认输入。
  • PromptCancel: prompt类型弹出框的取消事件，表示用户取消输入。

方法：

  • open():void: prompt类型弹出框的确认事件，表示用户确认输入。
  • alert(msg:string):void: alert类型弹出框的启动方法。
  • prompt():Promise: prompt类型弹出框的启动方法。
  • close():void: 弹出框的关闭方法。

Slot:

  • title: custom类型弹出框的title部分。
  • body: custom类型弹出框的body部分。
  • footer: custom类型弹出框的footer部分。

示例：

</wc-dialog>

时间选择器组件

属性：

  • type: 支持date,time,datetime三种选择模式，默认为date。date是日期选择器，time是时间选择器，datetime是时间日期选择器。
  • size: 支持big,normal,small三种弹出框，分别设置不同的时间选择框大小。
  • value: 设置选择器的初始值，这个字符串可以被Date对象格式化（new Date(datestring)）。比如：'2020 02/20 17:12'。
  • range: 设置选择器的选择范围 'datestring - datestring'，这个字符串可以被Date对象格式化（new Date(datestring)）,比如：'2020 02/20 00:00 - 2020 02/21 23:59'。
    （注意在设置range的时候要注意时和分的处理，即使在date类型的选择器下时和分也是客观存在的，要使整天可选，最好把range最后一天的时分位定义为23:59）

内部属性

  • value: 选中的日期，是一个Date类型的值，表示当前选中的日期。
  • datePickerValue: 日期选择器显示的日期，是一个Date类型的值，表示当前选择区的基准日期。

事件：

  • DateChange: 选中的日期发生变化，{detail:value}
  • DatePickerValueChange: 选择器中的显示日期发生变化。
  • Open: 选择框打开事件。
  • Close: 选择框关闭事件。

示例：

代码：

<wc-datetimepicker size="big" type="datetime" range="2022 06/25 00:00 - 2022 07/23 23:59" value="2022 06/25 08:10"></wc-datetimepicker>

AuthenticationForm组件

AuthenticationForm组件位于scripts/utils.js内，与input,timepicker，selector等组件可以交互组成表单结构。

AuthenticationForm类：

属性：

  • _formElements: 表单对象绑定的对象数组。每个元素对应组件的formElement属性。

方法：

  • isValid(): 检查表单绑定元素是否通过验证。返回true或false。
  • getValueByFieldName(fieldName): 获取与fieldName对应组件的值。
  • reset(): 重置整个表单实例。danger！
  • registerFormElement(formElement): 向表单实例里注册表单组件的formElement，组件元素内部使用。
  • clearForm(): 重置表单内组件元素到初始值。
